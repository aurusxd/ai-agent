# AI Agent Telegram Bot

Telegram-бот с ИИ-агентом на базе DeepSeek, который умеет самостоятельно выбирать и использовать внешние инструменты для выполнения запросов пользователя.

Агент может:

* отвечать на обычные вопросы;
* искать актуальную информацию в интернете;
* получать текущую погоду;
* выполнять математические вычисления;
* взаимодействовать с пользователем через Telegram;
* предоставлять HTTP API для интеграции с другими приложениями.

## Возможности

### ИИ-ответы

Агент использует DeepSeek через OpenAI-совместимый API и самостоятельно определяет, требуется ли для ответа внешний инструмент.

### Поиск в интернете

Для вопросов, связанных с новостями, ценами, событиями, компаниями и другой актуальной информацией, агент выполняет веб-поиск через Tavily API.

### Погода

Агент получает текущие погодные данные через OpenWeatherMap:

* температуру;
* ощущаемую температуру;
* описание погоды;
* скорость ветра.

### Калькулятор

Для математических запросов агент может вызвать отдельный инструмент вычислений.

### Telegram-интерфейс

Пользователь взаимодействует с агентом через обычного Telegram-бота. Все сообщения передаются в FastAPI-бэкенд, обрабатываются агентом и возвращаются пользователю.

### REST API

Проект предоставляет endpoint:

```http
POST /ask
```

Пример запроса:

```json
{
  "text": "Какая сейчас погода в Москве?"
}
```

Пример ответа:

```json
{
  "answer": "Погода в Москве..."
}
```

## Установка

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/aurusxd/ai-agent.git
cd ai-agent
```

### 2. Установите uv

```bash
pip install uv
```

### 3. Установите зависимости

```bash
uv sync
```

Также зависимости можно установить через стандартный `pip`:

```bash
pip install aiogram aiohttp fastapi loguru openai python-dotenv requests uvicorn
```

## Настройка

Создайте файл `.env` в корне проекта.

Можно скопировать пример:

### Windows

```bash
copy .env.example .env
```

### Linux и macOS

```bash
cp .env.example .env
```

Заполните переменные окружения:

```env
DEEPSEEK_API_KEY=your_deepseek_api_key
BOT_TOKEN=your_telegram_bot_token
TAVILY_API_KEY=your_tavily_api_key
WEATHER_API_KEY=your_openweathermap_api_key
```

### Где получить ключи

* `DEEPSEEK_API_KEY` — в личном кабинете DeepSeek.
* `BOT_TOKEN` — у `@BotFather` в Telegram.
* `TAVILY_API_KEY` — в личном кабинете Tavily.
* `WEATHER_API_KEY` — в личном кабинете OpenWeatherMap.

Не добавляйте файл `.env` в Git и не публикуйте API-ключи.

## Запуск

Для работы Telegram-бота необходимо запустить два процесса:

1. FastAPI-бэкенд.
2. Telegram-бот.

### 1. Запуск API

В первом терминале выполните:

```bash
uv run uvicorn app.api.main:app --host 127.0.0.1 --port 8000
```

После запуска документация API будет доступна по адресу:

```text
http://127.0.0.1:8000/docs
```

### 2. Запуск Telegram-бота

Во втором терминале выполните:

```bash
uv run python main.py
```

После успешного запуска отправьте боту команду:

```text
/start
```

## Проверка API

Пример запроса через `curl`:

```bash
curl -X POST "http://127.0.0.1:8000/ask" \
  -H "Content-Type: application/json" \
  -d "{\"text\":\"Что нового произошло в сфере ИИ?\"}"
```

Пример на Python:

```python
import requests

response = requests.post(
    "http://127.0.0.1:8000/ask",
    json={"text": "Какая сейчас погода в Москве?"},
    timeout=60,
)

response.raise_for_status()
print(response.json()["answer"])
```

## Как добавить новый инструмент

Чтобы расширить возможности агента:

1. Создайте новый Python-модуль в `app/tools/`.
2. Реализуйте функцию инструмента.
3. Добавьте описание функции в список `tools` в `app/agent/core.py`.
4. Зарегистрируйте функцию в словаре `available_tools`.
5. При необходимости добавьте новый API-ключ в `.env.example`.

Пример:

```python
def get_exchange_rate(currency: str) -> str:
    return f"Текущий курс для {currency}"
```

После регистрации агент сможет самостоятельно вызывать инструмент, когда он потребуется для ответа.
